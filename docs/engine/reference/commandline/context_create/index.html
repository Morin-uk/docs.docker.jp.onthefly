docker context create

読む時間の目安: 2 分

説明

コンテキストを生成します。

利用方法

docker context create [オプション] CONTEXT

追加説明

新たな context を生成ます。 これによって、別のクラスターや単一ノードへの接続を行う CLI 設定を、すばやく切り替えられるようになります。

本コマンドの利用例については、以下に示す 利用例の節 を参照してください。

オプション

名前／省略形 デフォルト 説明
--default-stack-orchestrator 指定されたコンテキストに対して利用するデフォルトのスタックオーケストレーターを設定します。(swarm|kubernetes|all)
--description コンテキストの内容説明を設定します。
--docker Docker のエンドポイントを設定します。
--from 既存のコンテキストから新たなコンテキストを生成します。
--kubernetes Kubernetes のエンドポイントを設定します。

利用例

Docker と Kubernetes エンドポイントを使ったコンテキストの生成

コンテキストを一から生成するには Docker オプションと、必要に応じて Kubernetes オプションを用います。 以下の例では、コンテキストmy-contextを生成し、Docker のエンドポイントとして/var/run/docker.sock、Kubernetes の設定ファイルとして/home/me/my-kube-configを指定します。

$ docker context create \
    --docker host=unix:///var/run/docker.sock \
    --kubernetes config-file=/home/me/my-kube-config \
    my-context

Create a context based on an existing context

既存のコンテキストを使って新たなコンテキストを生成するには、オプション --from=<コンテキスト名> を使います。 以下の例では、既存の existing-context というコンテキストから、新たに my-context というコンテキストを生成します。

$ docker context create --from existing-context my-context

--from オプションの指定がない場合、現在のコンテキストから新たなコンテキストが生成されます。

$ docker context create my-context

既存の DOCKER_HOST 環境変数を利用したスクリプトを使って、コンテキストを生成することもできます。

$ source my-setup-script.sh
$ docker context create my-context

既存のコンテキストから docker エンドポイントのみを参照して生成する場合は、--docker from=<コンテキスト名> オプションを使います。 以下の例では、my-context という新しいコンテキストを生成するにあたって、既存のコンテキスト existing-context から Docker エンドポイントを利用し、Kubernetes 設定は設定ファイル /home/me/my-kube-config から利用します。

$ docker context create \
    --docker from=existing-context \
    --kubernetes config-file=/home/me/my-kube-config \
    my-context

既存のコンテキストから kubernetes 設定のみを参照して生成する場合は、--kubernetes from=<コンテキスト名> オプションを使います。 以下の例は、my-context という新しいコンテキストを生成するにあたって、既存のコンテキスト existing-context から Kuberentes 設定を利用し、Docker エンドポイントは /var/run/docker.sock を利用します。

$ docker context create \
    --docker host=unix:///var/run/docker.sock \
    --kubernetes from=existing-context \
    my-context

Docker エンドポイントや Kubernetes 設定、またデフォルトのオーケストレーターや内容説明は、docker context updateを使って編集することができます。

Refer to the docker context update reference for details.

上位コマンド

コマンド 説明
docker context コンテキストを管理します。

関連コマンド

コマンド 説明
docker context create コンテキストを生成します。
docker context export コンテキストを tar ファイル、または Kubeconfig ファイルにエクスポートします。
docker context import コンテキストを tar ファイル、または zip ファイルからインポートします。
docker context inspect コンテキストに対する詳細情報を表示します。
docker context ls コンテキストを一覧表示します。
docker context rm コンテキストを削除します。
docker context update コンテキストを更新します。
docker context use カレントな Docker コンテキストを設定します。
