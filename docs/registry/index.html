Docker Registry

読む時間の目安: 1 分

このページでは、オープンソースである Docker Registry を使って、独自にレジストリを提供する情報を示しています。 Docker Hub はホスト提供込みの Registry であり、さまざまな機能を追加することができます。 たとえばチーム、組織、ウェブフック、自動ビルドなどです。 詳しくは Docker Hub を参照してください。

これは何か

Registory とは、ステートレスで拡張性の高いサーバーサイドのアプリケーションであり、Docker イメージを保存し提供するものです。 Registry はオープンソースであり Apache ライセンス のもとに提供されています。

なぜ利用するのか

Registry は以下の状況において利用することができます。

  • イメージの保存場所を厳密に管理したい場合。
  • 独自のイメージ配布場所を確保したい場合。
  • イメージの保存と配布を、社内の開発ワークフローに緊密に連携したい場合。

他の選択肢

メンテナンス不要ですぐに活用できるソリューションを求めるユーザーであれば、 Docker Hub をお勧めします。 これは無償で利用できる、ホスト提供込みの Registry であり、追加機能がさまざまにあります（組織アカウント、自動ビルドなど）。

システム要件

Registry は Docker engine バージョン 1.6.0 またはそれ以降 と互換性があります。

基本的なコマンド

レジストリを起動します。

docker run -d -p 5000:5000 --name registry registry:2

Docker Hub からイメージをプル（またはビルド）します。

docker pull ubuntu

イメージにタグづけを行って、レジストリを指し示すようにします。

docker image tag ubuntu localhost:5000/myfirstimage

イメージをプッシュします。

docker push localhost:5000/myfirstimage

もう一度プルします。

docker pull localhost:5000/myfirstimage

レジストリを停止して全データを削除します。

docker container stop registry && docker container rm -v registry
